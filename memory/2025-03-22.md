## 🧠 技術學習

### 1. PDF 處理完整流程
```
PDF → pdf2image → PIL Image → pytesseract OCR → 文字提取
```
- **關鍵依賴**: poppler (Windows), pdf2image, pytesseract
- **語言包**: chi_tra+eng (繁體中文+英文)
- **解析度**: 200 DPI 平衡速度與準確度

### 2. OCR 優化策略
| 方法 | 準確率 | 適用場景 |
|------|--------|----------|
| 純影像識別 | 60-70% | 一般發票 |
| 檔名輔助提取 | 95%+ | 租單、帳單 |
| 混合模式 | 90%+ | 推薦使用 |

**關鍵洞察**: 發票號碼從檔名提取遠比 OCR 準確

### 3. 去重機制設計
- **識別方式**: MD5(檔案路徑 + 修改時間 + 大小)
- **記錄期**: 7天
- **清理策略**: 自動過期清理

### 4. Excel/Google Sheets 相容
- **問題**: 日期自動轉換為序列號 (如 46103.49236)
- **解決**: 前導符號 `'` 強制文字格式
- **應用**: `'2025-09-15 10:00:00`

### 5. 分類邏輯強化
```python
if '租單' in filename or '租' in text:
    category = '42 - Rent'
elif '電' in text or 'CLP' in text:
    category = '37 - Operating General - Electricity, Water Supply'
```

## ⚠️ 錯誤與修正

| 問題 | 原因 | 修正 |
|------|------|------|
| 金額 $9 而非 $200,000 | 逗號分隔符導致解析失敗 | 改進正則: `[\d]{1,3}(?:,\d{3})+` |
| 重覆輸入 | 無去重機制 | 加入 Tracker 檔案追蹤 |
| 發票號碼錯誤 | OCR 不準確 | 從檔名提取 |
| 時間顯示數字 | Excel 自動轉換 | 加前導符號 `'` |
| API 配額耗盡 | 依賴 Google Vision | 改用本地 pytesseract |

## 🎯 方法論沉澱

### 第一原則應用
1. **能本地，不雲端** - 減少 API 依賴
2. **能檔名，不 OCR** - 結構化資訊優先
3. **小步測試，再批量** - 避免大規模錯誤
4. **防呆設計** - 預設重覆、格式問題

### 最佳實踐
- 圖片: pytesseract (本地)
- PDF: pdf2image + pytesseract
- 去重: 檔案雜湊 + 時間窗口
- 分類: 檔名關鍵字 > 內容分析
