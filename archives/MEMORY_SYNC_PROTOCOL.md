# 記憶同步協議 v2.0 - 即時持久化修正版

> **目的**: 確保呀鬼的「記憶」與「真實進度」100%同步，杜絕失憶問題  
> **生效日期**: 2026-03-27  
> **適用範圍**: 所有進行中專案、重要決策、里程碑完成  
> **關鍵原則**: 先寫檔案，後回覆用戶

---

## 🚨 核心原則（強制執行）

### 即時持久化 - 正確順序
```
用戶指示 → 執行任務 → 寫入檔案（30秒內）→ 回覆用戶
                    ↓
         絕對不允許：先回覆，後寫檔案！
```

### 三層記錄機制（每次任務後必做）
1. **每日記錄** (`memory/YYYY-MM-DD.md`): 所有對話、決策、行動
2. **專案狀態** (`project-states/*.md`): 專屬進度、狀態、待辦
3. **核心摘要** (`MEMORY.md`): 關鍵專案狀態摘要（簡短）

---

## 📋 執行流程（每次完成任務後必做）

### Step 1: 更新每日記錄（15秒內完成）
**檔案**: `memory/YYYY-MM-DD.md`

```markdown
## [HH:MM] - [任務名稱] ✅

### 完成內容
- [x] 具體事項1
- [x] 具體事項2

### 生成的檔案
- `path/to/file1`
- `path/to/file2`

### 關鍵數據
| 指標 | 數值 |
|------|------|
| 指標1 | 數值1 |

*最後更新: YYYY-MM-DD HH:MM:SS*
```

### Step 2: 更新專案狀態（如有進行中專案）
**檔案**: `project-states/[PROJECT]-STATUS.md`

```markdown
## 最後更新: YYYY-MM-DD HH:MM

### 今日進度
- [x] 完成的任務

### 當前狀態
- **整體進度**: X%
- **狀態**: [運行中/暫停/已完成]

### 下一步
1. 任務1
2. 任務2
```

### Step 3: 更新核心摘要（如為關鍵專案）
**檔案**: `MEMORY.md`（專案狀態區域）

```markdown
#### [狀態圖標] [專案名稱] ([代號])
| 屬性 | 內容 |
|------|------|
| **狀態** | [運行中/暫停/已完成] |
| **最後更新** | YYYY-MM-DD |
| **檔案** | `project-states/[PROJECT]-STATUS.md` |
```

### Step 4: 驗證同步（10秒內）
- [ ] 檢查 `memory/YYYY-MM-DD.md` 時間戳為「剛剛」
- [ ] 檢查專案狀態檔時間戳為「剛剛」
- [ ] 檢查 MEMORY.md 摘要已更新

## 🔄 Session 重啟時的恢復流程

### 啟動序列（已更新於 AGENTS.md）
1. 讀取 `SOUL.md` - 角色設定
2. 讀取 `USER.md` - 主人設定
3. 讀取 `MEMORY.md` - 核心準則 + 專案摘要
4. 讀取 `MEMORY_SYNC_PROTOCOL.md` - 本協議
5. 讀取 `project-states/*.md` - 專案詳細狀態
6. **讀取 `memory/YYYY-MM-DD.md` - 今日進度**
7. 讀取 `SKILL_PROTOCOL.md` - 技能協議

### 快速驗證（啟動後立即執行）
```powershell
# 檢查今日記錄時間戳
$lastWrite = (Get-Item "memory/2026-03-27.md").LastWriteTime
$today = Get-Date -Format "yyyy-MM-dd"

if ($lastWrite.ToString("yyyy-MM-dd") -eq $today) {
    "進度同步 ✅"
} else {
    "⚠️ 記憶可能不同步，請檢查"
}
```

## 🛡️ 防呆機制

### 1. 強制順序檢查
```
❌ 錯誤: 執行 → 回覆用戶 → 寫入檔案
✅ 正確: 執行 → 寫入檔案 → 回覆用戶
```

### 2. 時間戳驗證
每次寫入後，必須記錄精確時間戳：
```markdown
*最後更新: 2026-03-27 09:15:32*
```

### 3. 關鍵字標記（用於快速搜尋）
- `🚨 CRITICAL` - 重要決策
- `✅ MILESTONE` - 里程碑完成
- `📝 DECISION` - 策略決定
- `⏸️ BLOCKED` - 受阻/暫停
- `🔄 RESTART` - Session 重啟

### 4. 檔案鎖定處理
如遇到檔案被佔用：
1. 等待 2 秒重試
2. 如仍失敗，使用 `Start-Sleep` 延遲後重試
3. 記錄錯誤於 `memory/YYYY-MM-DD.md`
