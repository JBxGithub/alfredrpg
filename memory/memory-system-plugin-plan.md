## 🎯 專案目標

建立 OpenClaw 記憶系統插件，解決 Session Context 限制導致的「失憶」問題，實現多 Session 之間的無縫記憶傳承。

---

## 📋 背景與動機

### 現有問題
- OpenClaw 單一 Session Context 限制：256k tokens
- 當前 Session 使用率：75%（約 191k/256k）
- 每次開新 Session，AI 無法自動記得舊 Session 的細節對話
- 雖有 MEMORY.md 和每日記錄，但需要手動讀取，連續性不足

### 核心需求
靚仔希望建立一個系統，讓 AI 能夠：
1. 自動記錄對話內容（不只是行動清單）
2. 自動讀取相關記憶（不只是檔名搜索）
3. 保持決策過程和情感脈絡的連續性

## 🔧 核心功能模組

### 1. Session 監控模組
**職責**: 監控 Context 使用率，自動觸發記錄流程

**功能**:
- 每 5 分鐘檢測一次 Context 使用率
- 達到 70% 時發送預警
- 達到 80% 時自動觸發摘要生成
- 生成「Session 切換建議」給用戶

**輸出**:
```json
{
  "session_id": "agent:main:whatsapp:...",
  "context_usage": "75%",
  "tokens_used": 191000,
  "tokens_remaining": 65000,
  "status": "warning",
  "action": "建議開新 Session",
  "summary_ready": true
}
```

---

### 2. 自動摘要生成器
**職責**: 將對話內容轉換為結構化摘要

**摘要內容**:
- **決策記錄**: 做了什麼決定、為什麼這樣決定
- **行動項目**: 完成了什麼、待完成什麼
- **思考過程**: 為什麼選擇 A 而不是 B
- **情感脈絡**: 用戶的情緒狀態、關注點
- **關鍵數據**: 重要數字、配置、密碼（加密存儲）

**摘要格式**:
```markdown
## Session 摘要 [YYYY-MM-DD HH:MM]

### 對話主題
[主題標籤1, 主題標籤2, ...]

### 關鍵決策
1. [決策內容] - 原因：[原因]
2. ...

### 行動項目
- ✅ [已完成項目]
- ⏳ [待完成項目]

### 思考過程
[關鍵思考點]

### 情感脈絡
[用戶情緒狀態]

### 重要數據
- [數據項]: [數據值]

### 下次繼續
[建議下次對話的切入點]
```

---

### 3. 向量數據庫
**職責**: 長期存儲對話記憶，支援語義檢索

**技術選型**:
- **推薦**: ChromaDB（開源、本地部署、無需 API Key）
- **備選**: Pinecone（雲端、性能更好、需 API Key）

**數據結構**:
```python
{
  "id": "session_20260407_2245",
  "timestamp": "2026-04-07T22:45:00",
  "session_key": "agent:main:whatsapp:...",
  "content": "摘要內容",
  "embedding": [0.1, 0.2, ...],  # 向量嵌入
  "metadata": {
    "topics": ["交易策略", "MTF整合"],
    "decisions": ["保留3個策略", "清理2個策略"],
    "emotions": "積極、專注",
    "priority": "high"
  }
}
```

---

### 4. 記憶注入模組
**職責**: 新 Session 啟動時，自動讀取相關記憶

**流程**:
1. 新 Session 啟動
2. 檢測當前對話主題
3. 向量數據庫語義檢索（Top 5 相關記憶）
4. 生成「記憶上下文」注入對話
5. AI 基於記憶上下文回應

**注入格式**:
```markdown
[系統提示: 記憶恢復]

根據之前的對話記錄，以下是相關背景：

## 上次對話摘要 [時間]
[摘要內容]

## 關鍵決策
- [決策1]
- [決策2]

## 待完成項目
- [項目1]
- [項目2]

## 用戶偏好提醒
- [偏好1]
- [偏好2]

[繼續對話...]
```

| 項目 | 現有方案 (MEMORY.md) | 新方案 (記憶插件) |

| 用戶操作 | 需要手動管理 | 全自動 |
