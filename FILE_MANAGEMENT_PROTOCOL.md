# 子代理檔案管理規範

> 強制執行，防止亂建檔案

---

## 🚫 絕對禁止

1. **禁止喺根目錄亂建檔案**
   - ❌ 唔可以：`temp.py`, `test.txt`, `backup/`
   - ✅ 應該：放喺適當子目錄

2. **禁止重複命名**
   - ❌ 唔可以：`clawtrade-pro/` + `ClawTradePro/` + `clawtrade_pro/`
   - ✅ 應該：統一命名規範

3. **禁止跨目錄重複**
   - ❌ 唔可以：`skills/X/` + `private skills/X/`
   - ✅ 應該：技能只放一個地方

4. **禁止無文檔建檔案**
   - ❌ 唔可以：建咗檔案但唔更新 SKILL.md
   - ✅ 應該：建檔案同時更新文檔

---

## ✅ 正確流程

### 子代理任務前必須：

1. **讀取 SKILL_PROTOCOL.md**
   ```
   確認：
   - 目標位置
   - 命名規範
   - 現有檔案（避免重複）
   ```

2. **檢查現有檔案**
   ```powershell
   # 必須執行
   Get-ChildItem -Path "目標位置" -Name
   ```

3. **報告將創建嘅檔案**
   ```
   我將創建：
   - [ ] 檔案1: 位置 + 用途
   - [ ] 檔案2: 位置 + 用途
   ```

4. **完成後提交清單**
   ```
   已創建：
   - ✅ 檔案1: 完整路徑
   - ✅ 檔案2: 完整路徑
   ```

---

## 📁 檔案位置規則

| 類型 | 正確位置 | 錯誤示例 |
|------|---------|---------|
| 官方技能 | `skills/[skill-name]/` | ❌ `private skills/`, ❌ `projects/` |
| 私人技能 | `private skills/[skill-name]/` | ❌ `skills/`, ❌ `projects/` |
| 完整專案 | `projects/[project-name]/` | ❌ `skills/`, ❌ 根目錄 |
| 記憶檔案 | `memory/[YYYY-MM-DD].md` | ❌ `docs/`, ❌ 根目錄 |
| 文檔 | `docs/` 或專案內 `docs/` | ❌ 根目錄亂放 |
| 臨時檔案 | `/tmp/` 或專案內 `temp/` | ❌ 根目錄 |

---

## 🔍 定期審計清單

### 每日檢查：
- [ ] 根目錄有無新臨時檔案
- [ ] 有無重複命名

### 每週檢查：
- [ ] skills/ vs private skills/ 重複
- [ ] projects/ vs skills/ 重複
- [ ] 遺留檔案清理

### 每月檢查：
- [ ] 完整結構審計
- [ ] 命名規範統一
- [ ] 無用檔案清理

---

## 🛡️ 防護機制

### 子代理創建檔案前必須問：

1. **呢個檔案應該放邊？**
   → 查 SKILL_PROTOCOL.md

2. **會唔會同現有檔案重複？**
   → 執行 `Get-ChildItem` 檢查

3. **命名係咪符合規範？**
   → 小寫、連字符、無空格

4. **有無更新文檔？**
   → SKILL.md 必須同步更新

---

## ⚠️ 違規處理

發現亂建檔案：
1. 立即停止該子代理
2. 記錄問題到 `memory/skill-usage-log.md`
3. 清理亂建檔案
4. 重新指派任務
