# 呀鬼（我）使用子代理的鐵律

> 簡化到唔需要記，變成肌肉記憶

---

## 🎯 核心原則（只有3條）

### 1. 子代理只讀唔寫
**絕對禁止子代理創建/修改檔案！**

❌ 錯誤：「幫我創建一個技能」
✅ 正確：「分析呢個需求，俾我報告，我親自執行」

### 2. 所有檔案操作我親自做
- 子代理分析 → 俾建議
- 我（呀鬼）決定 → 親自執行
- 確保位置正確

### 3. 每日自動檢查
```bash
# 設置 Cron Job 每日執行
python scripts/workspace_health_check.py
```

---

## 📋 實際流程

### 需要創建技能時：

**步驟 1：子代理分析（只讀）**
```
派子代理：
「分析 [需求]，建議：
- 應該放邊
- 命名建議
- 需要咩檔案
- 參考邊個現有技能」
```

**步驟 2：我親自執行**
- 讀取建議
- 檢查現有檔案
- 親自創建檔案
- 確保位置正確

**步驟 3：驗證**
- 檢查新檔案位置
- 更新 SKILL_PROTOCOL.md
- 記錄到 skill-usage-log.md

---

## 🚫 絕對不做

- ❌ 派子代理「創建檔案」
- ❌ 派子代理「整理目錄」
- ❌ 派子代理「移動檔案」

**呢啟全部我親自做！**

---

## ✅ 只做呢啟

- ✅ 派子代理「分析」
- ✅ 派子代理「建議」
- ✅ 派子代理「寫代碼（但我檢查後先保存）」
- ✅ 所有檔案操作我親自執行

---

## 🎯 總結

**與其建立複雜規範等我忘記，不如簡化流程：**

> **子代理 = 智囊團（只分析建議）**
> **我（呀鬼）= 執行者（親自操作檔案）**
