# 2026-04-24 教訓 - Git 操作安全協議

## 🚨 事件回顧

**時間**: 2026-04-24 07:00  
**事件**: AlfredRPG 圖片意外被刪除  
**原因**: 使用 `git reset --hard origin/main` 覆蓋未 commit 檔案

## ❌ 錯誤行為

```bash
# 絕對禁止在未備份前執行
git reset --hard origin/main
git checkout -f main
```

呢啲指令會**永久刪除**所有未 commit 嘅更改，包括：
- 新檔案
- 修改中嘅檔案
- 圖片、數據檔案等

## ✅ 正確做法

### 1. 切換分支前必須備份
```bash
# 檢查未 commit 檔案
git status

# 備份重要檔案
cp -r important-files/ ../backup/

# 或者 stash
git stash push -m "backup before switching"
```

### 2. 安全切換分支流程
```bash
# Step 1: 檢查狀態
git status

# Step 2: 如有未 commit 檔案，先 stash
git stash

# Step 3: 切換分支
git checkout main

# Step 4: 還原 stash
git stash pop
```

### 3. 強制重置前必須確認
```bash
# 絕對禁止直接執行
git reset --hard

# 正確流程
git status  # 檢查有無重要未 commit 檔案
git stash    # 先備份
git reset --hard origin/main
git stash pop  # 嘗試還原
```

## 📝 檢查清單

執行任何破壞性 Git 操作前，必須確認：
- [ ] 已檢查 `git status`
- [ ] 已備份所有未 commit 檔案
- [ ] 已通知用戶並獲得批准
- [ ] 已記錄操作原因

## 🔴 絕對禁止行為 (2026-04-24 新增)

1. **未經確認執行 `git reset --hard`**
2. **未備份前執行 `git checkout -f`**
3. **未檢查狀態直接強制操作**

## 💡 教訓

> 「小心駛得萬年船」— 任何可能影響用戶數據嘅操作，必須先備份、後執行。
