## 🔄 數據流

```
交易日:
├─ 09:30 開盤
│  ├─ Futu OpenD 獲取實時數據
│  ├─ 計算 Absolute/Reference Score
│  └─ 生成交易信號
│
├─ 交易時段
│  ├─ 監控持倉 (止損/止盈/減倉)
│  ├─ 執行開倉/平倉
│  └─ 記錄交易
│
└─ 16:00 收盤
   ├─ 計算每日績效
   ├─ 檢查成就解鎖
   └─ 保存數據到數據庫

非交易日:
├─ 08:00 每日監控
│  └─ 檢查異常、發送警報
│
├─ 週一 09:00 每週回顧
│  └─ 分析趨勢、生成建議
│
└─ 每月1日 10:00 每月調整
   └─ 評估參數、自動調整 (可選)
```

## 🛡️ 安全機制

### 多層保護

```
第一層: 每日監控
├─ 單日虧損 > $2000 → 警告
├─ 連續3天虧損 → 自動減倉50%
└─ 單日回撤 > 15% → 暫停24小時

第二層: 每週回顧
├─ 回撤 > -70% → 建議減倉至85%
├─ 勝率 < 60% → 建議提高入場門檻
└─ 盈利減倉效率低 → 調整門檻

第三層: 每月調整
├─ 平均回撤 < -65% → 自動減倉
├─ 平均勝率 < 65% → 自動提高門檻
└─ 表現優秀 → 可適度提高倉位
```

### 參數保護

| 參數 | 自動調整 | 需確認 | 保護級別 |
|------|----------|--------|----------|
| max_position_pct | ≤5% | >5% | 高 |
| profit_level_1/2 | ✅ | - | 中 |
| long_stop/short_stop | ❌ | ✅ | 極高 |

## 🚀 部署建議

### 階段部署

**階段 1: 觀察期 (第1-2週)**
- 啟用所有監控
- `auto_adjust: false`
- 每日審閱報告
- 熟悉系統行為

**階段 2: 測試期 (第3-4週)**
- `confirm_before_adjust: true`
- 對每個建議進行回測驗證
- 評估建議準確率

**階段 3: 自動期 (第5週+)**
- 可考慮啟用 `auto_adjust: true`
- 持續監控自動調整效果
- 定期回顧長期趨勢

### 緊急情況處理

```
情況: 系統異常/黑天鵝事件
行動:
1. 立即暫停交易引擎
2. 檢查持倉狀態
3. 手動平倉 (如必要)
4. 分析異常原因
5. 調整參數後重新啟動
```

### 常見問題

**Q: 系統無法啟動？**
A: 檢查數據庫連接、Futu OpenD 狀態、Node-RED 運行狀態

**Q: 沒有生成交易信號？**
A: 檢查評分計算是否正常、閾值設置是否過嚴

**Q: 建議質量不高？**
A: 延長觀察期，調整配置中的閾值
