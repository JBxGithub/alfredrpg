## 一、系統概述

### 1.1 什麼是六循環系統？

六循環系統是一個**自動化交易決策系統**，專門用於分析納斯達克100指數（NQ 100）並交易相關ETF（TQQQ）。

### 1.2 核心功能

| 功能模塊 | 說明 | 自動化程度 |
|---------|------|-----------|
| 數據收集 | 從富途、Yahoo等獲取實時市場數據 | 全自動 |
| 技術分析 | 計算多維度技術指標（Absolute + Reference評分） | 全自動 |
| 交易決策 | 生成買入/賣出/持有信號 | 全自動 |
| 風險控制 | 止損止盈、倉位管理、日虧損限制 | 全自動 |
| 績效追蹤 | 記錄每日表現、成就徽章系統 | 全自動 |
| 策略優化 | 每週自動回顧並優化參數 | 全自動 |

### 1.3 分析標的與交易標的

| 類型 | 標的 | 說明 |
|------|------|------|
| **分析標的** | NQ 100 (納斯達克100指數) | 使用QQQ作為數據代理（相關性>99%） |
| **交易標的** | TQQQ (ProShares UltraPro QQQ) | 3倍槓桿納斯達克100 ETF |

## 二、系統架構

### 2.1 六循環流程

```
感知層 → 處理層 → 決策層 → 執行層 → 成就層 → 學習層 → (循環)
```

| 層級 | 功能 | 技術實現 | 用戶參與度 |
|------|------|---------|-----------|
| **感知層** | 收集市場數據 | Futu OpenD API + Node-RED | 無需參與 |
| **處理層** | 計算技術指標 | Node-RED Flows | 無需參與 |
| **決策層** | 生成交易信號 | Python決策引擎 | 可查看信號 |
| **執行層** | 執行交易 | FutuTradingBot | 建議手動確認 |
| **成就層** | 記錄表現 | Alfredrpg系統 | 可查看報告 |
| **學習層** | 策略優化 | Self-Improving | 無需參與 |

### 2.2 數據源配置

| 數據源 | 優先級 | 狀態 | 備註 |
|--------|--------|------|------|
| Futu OpenD (QQQ) | 主要 | 運行中 | 免費版，穩定 |
| investing.com | 備份 | 待命 | NQ 100指數 |
| Yahoo Finance | 備份 | 待命 | ^NDX數據 |

## 三、風險管理

### 3.1 多層次風險控制

| 風險類型 | 控制措施 | 閾值 | 觸發後果 |
|---------|---------|------|---------|
| **單筆風險** | 最大倉位限制 | 1,000股 | 無法開新倉 |
| **日虧損限制** | 每日最大虧損 | $500 (2%) | 自動停止交易 |
| **總持倉風險** | 總持倉價值 | 4% | 自動減倉 |
| **止損保護** | 單筆止損 | 2% | 自動平倉 |
| **止盈保護** | 單筆止盈 | 3% | 自動平倉 |
| **錯誤保護** | 系統錯誤 | 任何錯誤 | 暫停一天 |

### 3.2 緊急停止程序

如遇突發情況，可立即執行以下命令停止系統：

```powershell
# 停止所有Python進程
Get-Process python | Stop-Process

# 記錄停止原因
echo "緊急停止: $(Get-Date) - [原因]" >> logs/emergency_stop.log
```

## 八、常見問題（FAQ）

**Q1: 系統會自動執行交易嗎？**  
A: 目前數據流和決策已完全自動化，但交易執行建議手動確認，以確保風險可控。

**Q2: 我需要每天監控系統嗎？**  
A: 不需要持續監控。建議每日開市前花30秒運行監控命令確認系統正常即可。

**Q3: 如果系統出現異常怎麼辦？**  
A: 系統設有多重保護機制：止損止盈、日虧損限制、錯誤自動暫停。您也可以隨時手動停止系統。

**Q4: 可以修改風險參數嗎？**  
A: 可以，但建議先與系統管理員討論，以確保風險可控。

**Q5: 系統的數據準確性如何？**  
A: 使用QQQ作為NQ 100指數的代理，兩者相關性超過99%，技術分析結果高度一致。
